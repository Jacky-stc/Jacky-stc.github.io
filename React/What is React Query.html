什麼是 React Query（TanStack Query）?​

參考資料

https://tanstack.com/query/latest

這是官方文件提供的說法

It makes fetching, caching, synchronizing and updating server state in your web applications a breeze.

大多數的web框架都沒有一套比較完善且固定的方法去處理data-fetching，其中你會遇到許多問題，例如error-handling、快取、資料處理等，在React中你也許會選擇透過 use state、use effect、try、catch等來解決error-handling的問題，下面是一個簡單的例子：

import React, { useState, useEffect } from 'react';

const DataFetchingComponent = () => {
  const [data, setData] = useState(null);
  const [error, setError] = useState(null);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    const fetchData = async () => {
      try {
        setLoading(true);
        const response = await fetch('https://api.example.com/data');
        if (!response.ok) {
          throw new Error(`HTTP error! status: ${response.status}`);
        }
        const result = await response.json();
        setData(result);
      } catch (err) {
        setError(err.message);
      } finally {
        setLoading(false);
      }
    };

    fetchData();
  }, []);

  if (loading) return <p>Loading...</p>;
  if (error) return <p>Error: {error}</p>;

  return <div>{JSON.stringify(data)}</div>;
};

但這樣會導致你的程式碼非常冗長且難以維護，於是開發者將一系列的功能整合在一起生成了Tanstack Query這個第三方套件，幫助我們以最簡潔的方式處理這些讓人頭痛的問題

warning

以下的操作都是以Next.js app router為主，可能會與單純使用React或是Next.js page router有些許差異

安裝（Installation）​

npm i @tanstack/react-query

配置（Setup）​

首先要先創立一個Provider將網站包裹起來，我們才能在裡面執行React-query的功能，但是由於 QueryClientProvider必須要在Client-Side執行，所以我們要另外創立一個Client Component Provider.tsx 並透過import將其導入layout.tsx中，而不能直接在layout.tsx中設定。 以下是React-Query官方的設定檔：

// In Next.js, this file would be called: app/providers.tsx
'use client'

// Since QueryClientProvider relies on useContext under the hood, we have to put 'use client' on top
import {
  isServer,
  QueryClient,
  QueryClientProvider,
} from '@tanstack/react-query'

function makeQueryClient() {
  return new QueryClient({
    defaultOptions: {
      queries: {
        // With SSR, we usually want to set some default staleTime
        // above 0 to avoid refetching immediately on the client
        staleTime: 60 * 1000,
      },
    },
  })
}

let browserQueryClient: QueryClient | undefined = undefined

function getQueryClient() {
  if (isServer) {
    // Server: always make a new query client
    return makeQueryClient()
  } else {
    // Browser: make a new query client if we don't already have one
    // This is very important, so we don't re-make a new client if React
    // suspends during the initial render. This may not be needed if we
    // have a suspense boundary BELOW the creation of the query client
    if (!browserQueryClient) browserQueryClient = makeQueryClient()
    return browserQueryClient
  }
}

export default function Providers({ children }: { children: React.ReactNode }) {
  // NOTE: Avoid useState when initializing the query client if you don't
  //       have a suspense boundary between this and the code that may
  //       suspend because React will throw away the client on the initial
  //       render if it suspends and there is no boundary
  const queryClient = getQueryClient()

  return (
    <QueryClientProvider client={queryClient}>{children}</QueryClientProvider>
  )
}
// In Next.js, this file would be called: app/layout.tsx
import Providers from './providers'

export default function RootLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (
    <html lang="en">
      <head />
      <body>
        <Providers>{children}</Providers>
      </body>
    </html>
  )
}

資料抓取（Data-Fetching）​

useQuery useQuery適用於任何非同步的資料請求，他會與一個unique key做綁定，有點類似useEffect中dependency的概念，要使用useQuery最少需要給予queryKey、queryFn兩個參數。

  • queryKey：queryKey是一組數列，每一個useQuery都應該綁定一組特定的queryKey
  • queryFn：一個回傳值是Promise的function，結果必須要是 resolve 或是 throw new Error
import { useQuery } from '@tanstack/react-query'

function App() {
  const info = useQuery({ queryKey: ['todos'], queryFn: fetchTodoList })
}

我們把 info打印出來就可以看到裡面包含了我們需要的data以及各種非常實用的參數，像是isLoading、isError、isFetching等等
