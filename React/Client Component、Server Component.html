React Client Component、React Server Component​

參考資料

https://github.com/reactwg/server-components/discussions/4 https://github.com/reactwg/server-components/discussions/5 https://react.dev/reference/rsc/server-components https://nextjs.org/learn/react-foundations/server-and-client-components https://www.webscope.io/blog/server-components-vs-ssr https://www.plasmic.app/blog/how-react-server-components-work#what-are-react-server-components https://blog.jankoritak.com/so-why-server-components

在2020年以前，React渲染元件的方式主要集中在客戶端，2020年末React推出了React Server Components（RSC），RSC希望能同時達成以下三個原則

  • 良好的用戶體驗（Good user experience）
  • 簡易的維護方式（Cheap maintainence）
  • 快速的載入速度（Fast performance）

在介紹Client Component與Server Component之前需要先了解網站中客戶端與伺服器端的交互方式

伺服器端與客戶端​

網站的運行環境：

  • 客戶端（Client）透過向伺服器端發送請求取得程式碼，並將回傳的程式碼轉換為用戶可以進行互動的頁面。
  • 伺服器端（Server）是儲存你所有網站程式碼的資料中心，當接收來自客戶端的請求時執行某些操作並給予適當的回應。

這兩個環境各自有它們的功能與限制，正是這樣的特性使得某些操作更適合在其中一個環境下執行，因此也就有了區分Client Component以及Server Component的需求。

warning

在React中，server與client並不是指稱實際物理意義上的「伺服器」與「瀏覽器」，你可以把他們想成是「React server」以及「React client」，Props永遠會從React server傳向React client，且兩者之間有著序列化邊界（Serialization Boundary），React server通常在build time或是實際的伺服器上運行，React client則通常在兩種環境下運行（在瀏覽器負責管理DOM，在其他環境負責生成初始化HTML）

Network Boundary​

Network Boudary是一個React用來區分React server以及React client之間的概念分界線，在React中你可以自行決定分界線所在的位置，當你在某個元件的頂層使用了 'use client'，系統便會將其視為server component與client component的邊界，該元件以及其所有的子元件都會被視為client component，因此如果你希望在某個子元件使用client component的功能，你只需要在父元件設置 'use client' 即可

渲染過程​

React server會優先將渲染Server Components，渲染完的Server Components會產生一個特殊的檔案格式叫做React Server Component payload（RSC payload），其中會夾帶以下資訊：

  • Server Component渲染完成的靜態HTML
  • Client Component在Server Component中的佔位符（placeholder）以及相對應的JS檔案路徑
  • 所有Server Component需要傳遞給Client Component的Props

接著會將這些資訊傳送給React client之後，React client會進行Client Component的靜態HTML渲染，最後將Server Component、Client Component的靜態HTML以及處理互動事件或是與瀏覽器相關的JS代碼等打包傳送至瀏覽器，瀏覽器端會先呈現靜態的HTML，並對Client Component進行Hydration使他們有互動功能

warning

React Client Component不是只能在Client-Side渲染

React的命名方式可能很容易讓人誤以為Client Component就是CSR，Server Component就是SSR，但其實Client Component的部分渲染過程還是在server side完成

Server Component的優點​

  • 可以直接跟伺服器拿資料，減少瀏覽器跟伺服器之間來回溝通的成本
  • Server Component具有Streaming機制，可以分段傳送渲染完成的靜態HTML，優化Time to first byte（TTFB）的時間
  • 減少傳送至瀏覽器的bundle js大小，由於server component是在server進行編譯，因此我們就不必將各類的函式庫，或是一些計算的function等等一起打包傳送至瀏覽器
