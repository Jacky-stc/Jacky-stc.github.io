Event loop​

JavaScript屬於**單執行緒（Single threaded）**的語言，表示一次只能處理一個任務(task)，當前任務完成之前不會執行下一個任務，這個概念稱之為同步（Synchronous）

同步（Synchronous）與非同步（Asynchronous）​

當你一次只能做一件事，這件事情沒有完成之前不能做其他事情就稱為同步（Synchronous）處理（聽起來確實蠻怪的），以煮飯來說，你必須要等到飯煮熟了才能去煮菜，水煮滾了你才能去切菜，可以想像這是多麼沒有效率的方式，如果你能在等飯煮熟的同時先不管他，同時去進行其他的工作，這就是非同步，或是異步（Asynchronous）處理

同步​

SYNCHRONOUS
   |--------A--------|
                     |--------B--------|

非同步​

ASYNCHRONOUS
   |--------A--------|
         |--------B--------|

JavaScript之所以能做到非同步這件事，必須歸功於事件迴圈(Event loop) 以及 執行環境(Execution environment)，執行環境是指JavaScript程式碼執行時的環境，這包括了JavaScript本身以及其所運行所依賴的元件或是外部環境，例如瀏覽器或是node.js，執行環境負責提供額外的功能，使JavaScript可以執行非同步處理以及其他超出其語言本身的能力

Event Loop 組成​

Event loop主要由Heap、Stack、Queue 組成

  • Heap (記憶體堆)
    • 負責管理應用程式中的物件和變數儲存
    • 是一個大型非結構化的記憶體空間，用於動態分配資料
  • Stack (堆疊)
    • 負責記錄目前正在執行中的函數
    • 採取先進後出(LIFO, Last in first out) 原則，最後進入Stack的函數會最先被執行
  • Queue (隊列)
    • 排列等待被執行的函數
    • 分成了Macrotask以及Microtask (其實只有Microtask，Macrotask只是因應Microtask而產生的說法)，Queue會優先執行所有Microtask的任務
    • 常見的Macrotask：setTimeOut、setInterval、I/O
    • 常見的Microtask：Promise.then()、queueMicortask、MutationObserver

Event loop 流程​

Event loop示意圖 (圖片擷取自YouTube頻道"Lydia Hellie")

  • JavaScript在執行過程中遇到函數就會呼叫Call Stack，將函數放到Stack中執行，當一個函數執行完畢就會從Stack頂部抽離，繼續執行下一個。
  • 遇到需要同步執行的函數時，執行環境就會呼叫相關的API (例如Web APIs中的setTimeOut)接著將他放進Task Queue (或是稱為Macrotask Queue、Callback Queue) 或是Micortask Queue中等待執行。
  • 當Stack中的函數都執行完畢之後，會優先執行Microtask Queue中的函數，執行完畢則會接著執行Task Queue (若在執行Task Queue中的函數時產生了新的Microtask，也會優先執行完Microtask再繼續執行下一個Task Queue中的函數)
  • 不斷重複以上循環直到所有的隊列為空，該過程就稱為Event loop
